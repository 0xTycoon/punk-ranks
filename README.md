### There are 10,000 punks. How was the top 500 list generated?


Let's find out!


Category Attributes
---
* Type (Alien 9, Ape 24, Zombie 88, Human/Male 6039, Human/Female 3840)
* Skin (Human/Mid 3031, Human/Dark 2824, Human/Light 3006, Human/Albino 1018)


Decorative attributes
---

Each punk can have 0 to 7 decorative attributes.

Meta-attributes
---

The number of decorative attributes a punk has is also an attribute itself. Here to referred as `Attribute Score`

Other meta-attributes may include the number of colors in the palette, number of pixels, "hidden attributes", etc.

There are also the "dev punks" which are numbered between 0 and 999

It's important to note that punks start at 0 - that means the 0th punk is the first. Also, some punks of
a particular number can be worth more, eg. punk id `6666` or `9999` and so on.

For brevity, only the `Attribute Score` meta-attribute is used in this project.

Methodology
----

A complete list of punks and their attributes was exported form a public Google Spreadsheet amd exported as `punks.csv` 

A Go program was prepared to import the data into MySQL

The distribution of the Types, Number Of Attributes and Accessories was tallied up.

A new column was added called `category` and `category_score`. 
The categories were grouped according to the tree shown under the *Taxonomy* heading.

A table, `attributes`  contains all the accessory attributes and tallies.

A table, `punk_attributes` contains all the links between the punks and their accessories


Next, the scores were re-computed.

The ranks.csv and ranks.md files were prepared from the result.

Finally, an SQL query was prepared to crunch the rankings. See the comments in the SQL query for more details.

The database has been exported to the [data.sql](data.sql) file.

The SQL Query
-----




```mysql

select
ROW_NUMBER() over () AS  ranking,
CONCAT("<img src=\"https://www.larvalabs.com/public/images/cryptopunks/punk",
   CONCAT(LPAD(id, 4, 0), ".png\" width=\"48\" height=\"48\">"))  punk_image,
CONCAT(concat('<a href=\"https://www.larvalabs.com/', concat('cryptopunks/details/',
   concat(LPAD(id, 4, 0), concat('">', concat(LPAD(id, 4, 0), '</a>'))))))  id,
minscore,                           -- minscore is the score of the best attribute
2nd,                                -- 2nd is the score of the 2nd best attribute. used to resolve ties.
category_score,
att_count_score,
(select group_concat(concat(a2.name, concat(' ', a2.total / 100)) ORDER BY punk_attributes.score)
 from punk_attributes
          join attributes a2 on punk_attributes.attribute_id = a2.id AND punk_attributes.punk_id = p.id
 group by punk_attributes.punk_id) attributes,
p.skin,
p.type,
CASE
    WHEN slots = 0
        -- 0-trait punks (8/10000)
        -- there a no attributes, just take the harmonic mean of the Category Score & Attribute Score
        -- (it produces a fairer value, leaning more to the lower
        -- number, since a lower number means it's more rare).
        -- eg, if the category score si 10.1800000000 and the Attribute Score is 0.0800000000
        -- harmonic mean is: 0.15875243664717
        -- arithmetic mean is: 5.13
        -- Thus you can see that the harmonic mean produces a better,
        -- lower, score that leans towards the rareness of the NFT
        -- Also: since 0-trait punks should really be in their own category, we scale them with a constant of 0.01
        -- The rationale for 0.01 was sp that it's in the same magnitude as the Apes & Zombies
        THEN 0.01 / (((1 / p.category_score) + (1 / p.att_count_score)) / 2)
    WHEN slots = 1
        -- for single attribute punks, same as above, but we include the minscore in Harmonic Mean score,
        -- then we take the reciprocal
        THEN 1 / (((1 / p.category_score) + (1 / p.att_count_score) + (1 / minscore)) / 3)
    WHEN slots > 1 and slots < 6
        -- 1-trait punks to 6-trait punks
        -- Take the harmonic mean of all scores including the attribute-count score
        -- Single attribute punks aren't as rare, but we also need to consider the Attribute Score
        -- to resolve ties, we also use the 2nd best (rarest) attribute in the mix
        THEN 1 / (((1 / p.category_score) + (1 / p.att_count_score) + (1 / minscore) + (1 / 2nd)) / 4)
    WHEN slots = 6
        -- 6-trait punks are extremely rare
        -- minscore is used as the numerator instead of using the 0.01 constant.
        -- This helps to disperse these punks's ranks
        THEN  minscore / (((1 / p.category_score) + (1 / p.att_count_score)) / 2)
    WHEN slots = 7 THEN
        -- 7-trait punk. (1/10000)
        -- This one is so badass, it's like it's in its own category
        -- computing the score based on attributes would not be fair,
        -- the number of attributes is what matters on this punk!
        -- We do not want to use minscore here, because it would not do it justice, a constant was chosen instead
        -- The rationale for 0.01 was to be consistent with the 0 attribute punks
        0.01 / (((1 / p.category_score) + (1 / p.att_count_score)) / 2)
    END
-- finally, we add some weights based on the scarcity of the non-human categories
    * IF(type = "Alien", 9 / 1000, 1)
    * IF(type = "Zombie", 88 / 1000, 1)
    * IF(type = "Ape", 24 / 1000, 1) total_score
from (select punks.*,
     min(punk_attributes.score) * 0.01 minscore, -- give slightly more weight to the best attribute!
     -- 2nd best score, some punks have <= 1 attributes so score will be null
     ifnull((select min(pa.score) -- lowest score excluding the lowest is the 2nd lowest.
             from punk_attributes pa
                      JOIN attributes a on pa.attribute_id = a.id AND pa.punk_id = punks.id
             where score
                   -- exclude the lowest score
                   NOT in (
                   select min(score)
                   from punk_attributes pa
                            JOIN attributes a on pa.attribute_id = a.id
                   WHERE pa.punk_id = punks.id
                   group by pa.punk_id
                   )
             group by punks.id
            ), min(score)) 2nd
  from punks
           join punk_attributes on punks.id = punk_attributes.punk_id
  group by punk_attributes.punk_id) p
order by total_score;

```
